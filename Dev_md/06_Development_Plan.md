## 🚀 향후 개발 계획 (Future Development Plans)

다음은 프로젝트의 안정성, 기능성 및 사용자 경험을 더욱 향상시키기 위한 제안된 개발 계획입니다.

### A. 성능 및 비용 최적화

1.  **LLM API 호출 캐싱 전략 도입**:
    *   **세부 작업**: 동일한 기사 URL에 대한 LLM(요약, 감성 분석) API 중복 호출을 방지하기 위해 백엔드 서비스에 캐싱 메커니즘을 구현합니다.
    *   **고려 사항**: 인메모리 캐시 (단기) 또는 `Redis` 같은 외부 캐시 저장소 (장기) 도입을 검토합니다.
    *   **목표**: LLM API 비용 절감 및 응답 시간 개선.

### B. 본문 스크래핑 견고성 강화

1.  **고급 웹 스크래핑 도구 도입 검토**:
    *   **세부 작업**: 현재 `requests`와 `BeautifulSoup`으로 수집하기 어려운 동적 콘텐츠(JavaScript 렌더링) 웹사이트를 위해 `Selenium` 또는 `Playwright`와 같은 브라우저 자동화 도구 도입을 검토합니다.
    *   **목표**: 뉴스 본문 추출 성공률 향상 및 지원 가능한 뉴스 소스 확대.

### C. 사용자 경험 및 기능 확장

1.  **분석 결과 저장 및 공유 기능**:
    *   **세부 작업**: 사용자가 분석한 뉴스 기사의 요약 및 감성 분석 결과를 저장하고, 다른 사용자와 공유할 수 있는 기능을 추가합니다.
    *   **고려 사항**: 사용자 인증 시스템, 데이터베이스 연동, 공유 링크 생성 기능 등을 검토합니다.
    *   **목표**: 사용자 편의성 증대 및 앱 활용성 확장.

2.  **뉴스 검색 필터링 및 정렬**:
    *   **세부 작업**: NewsAPI 검색 결과에 대해 날짜, 출처, 관련성 등 다양한 기준으로 필터링하고 정렬할 수 있는 기능을 프론트엔드에 추가합니다.
    *   **목표**: 사용자에게 더 세분화된 검색 옵션 제공.

### D. 배포 및 운영

1.  **HTTPS 적용 및 도메인 연결**:
    *   **세부 작업**: 프로덕션 환경 배포 시 HTTPS를 적용하여 데이터 통신 보안을 강화하고, 적절한 도메인을 연결합니다.
    *   **목표**: 보안 강화 및 사용자 신뢰도 향상.

2.  **모니터링 및 로깅 시스템 구축**:
    *   **세부 작업**: 백엔드 API의 성능, 에러 발생률, LLM API 사용량 등을 모니터링하고 로그를 수집하는 시스템을 구축합니다.
    *   **고려 사항**: Prometheus, Grafana, ELK Stack 등을 검토합니다.
    *   **목표**: 시스템 안정성 및 문제 발생 시 빠른 대응 능력 확보.
