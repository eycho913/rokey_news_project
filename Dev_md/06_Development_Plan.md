# 6. 다음 개발 계획 (To-Do 리스트)

이 문서는 현재 프로젝트 상태를 기준으로 식별된 개선, 수정 및 추가 개발 항목을 정리한 To-Do 리스트입니다.

## A. 핵심 기능 및 아키텍처 조정

1.  **[ ] 키워드 기반 뉴스 검색 기능 구현**:
    -   **세부 작업**: 백엔드에 키워드를 받아 NewsAPI로 뉴스를 검색하는 새로운 FastAPI 엔드포인트(예: `/search`)를 생성합니다.
    -   **세부 작업**: 프론트엔드(React)에 검색창을 추가하여 이 새로운 엔드포인트를 호출하고 검색 결과를 목록 형태로 표시합니다.
    -   **목표**: 사용자가 URL을 직접 입력하는 대신 키워드로 뉴스를 탐색할 수 있도록 하여 앱의 핵심 사용성을 확보합니다.

2.  **[ ] 아키텍처 방향성 확정 및 문서 통일**:
    -   **세부 작업**: 현재 코드 베이스(React/FastAPI)를 공식 아키텍처로 확정하고, Streamlit 기반으로 작성된 모든 개발 문서를 현행화합니다.
    -   **목표**: 프로젝트의 문서와 실제 구현의 불일치를 해소하여 유지보수성과 협업 효율성을 높입니다.

## B. 백엔드 및 API 개선

1.  **[ ] API 키 처리 방식 보안 강화**:
    -   **세부 작업**: 프론트엔드에서 API 키를 요청 본문에 담아 보내는 현재 방식 대신, 백엔드 서버가 직접 환경 변수에서 API 키를 로드하도록 수정합니다.
    -   **세부 작업**: `AnalyzeRequest` 모델에서 API 키 관련 필드를 제거합니다.
    -   **목표**: 민감한 API 키가 클라이언트에 노출되거나 네트워크를 통해 전송되지 않도록 하여 보안을 강화합니다.

2.  **[ ] LLM 공급자 설정 백엔드화**:
    -   **세부 작업**: LLM 공급자(Gemini, OpenAI 등) 및 모델 선택을 클라이언트 요청이 아닌 백엔드 설정(환경 변수 또는 설정 파일)으로 관리합니다.
    -   **목표**: 프론트엔드는 비즈니스 로직에만 집중하고, 백엔드 인프라 관련 설정을 분리하여 유연성을 확보합니다.

## C. 프론트엔드 개선

1.  **[ ] 누락된 UI 기능 구현**:
    -   **세부 작업**: 검색 결과(뉴스 목록)를 표시할 데이터 그리드 또는 리스트 UI를 구현합니다.
    -   **세부 작업**: 목록에서 특정 기사를 선택했을 때 요약 및 감성 분석 결과를 보여주는 상세 뷰를 구현합니다.
    -   **세부 작업**: 분석 결과를 JSON 또는 CSV 파일로 다운로드할 수 있는 버튼을 추가합니다.
    -   **세부 작업**: 백엔드 처리 시간 동안 사용자가 대기 상태를 인지할 수 있도록 로딩 스피너 또는 프로그레스 바를 구현합니다.

## D. 테스트 및 품질 보증

1.  **[ ] 테스트 스위트 구축**:
    -   **세부 작업**: `backend-api` 내부에 `tests/` 디렉토리를 생성하고, `pytest`를 사용하여 API 엔드포인트 및 서비스 로직에 대한 단위/통합 테스트를 작성합니다.
    -   **세부 작업**: `frontend`에 `Jest` 및 `React Testing Library`를 설정하고, 주요 컴포넌트에 대한 테스트를 작성합니다.
    -   **목표**: 코드 변경에 따른 부작용을 최소화하고, 애플리케이션의 안정성과 품질을 보장합니다.

2.  **[ ] CI/CD 파이프라인 업데이트**:
    -   **세부 작업**: `.github/workflows/ci.yml` 파일을 수정하여 프론트엔드와 백엔드의 의존성 설치, 린트, 테스트가 모두 자동 실행되도록 구성합니다.
    -   **목표**: 코드 품질을 일관되게 유지하고, 배포 전 잠재적 오류를 자동으로 검출합니다.
