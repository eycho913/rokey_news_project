# 3. 콘텐츠 및 프로젝트 성공 기준 (v2.1)

## 3.1. 목적
- 이 문서는 Rokey News Project에서 제공하는 AI 생성 콘텐츠(요약, 감성 분석)의 품질과 프로젝트의 성공 여부를 측정하기 위한 기준을 정의합니다.

## 3.2. AI 생성 콘텐츠 품질 기준
1.  **요약의 정확성**:
    - 요약문은 원본 기사의 핵심 내용을 왜곡 없이 포함해야 합니다.
    - 주요 사건, 인물, 결론이 누락되지 않아야 합니다.

2.  **요약의 가독성**:
    - 요약문은 문법적으로 정확하고 자연스러운 문장으로 구성되어야 합니다.
    - 불릿 포인트 등을 활용하여 사용자가 쉽게 이해할 수 있어야 합니다.

3.  **감성 분석의 신뢰성**:
    - 분석된 감성(긍정/부정/중립)은 일반적인 상식선에서 원본 기사의 논조와 일치해야 합니다.
    - 특히, 리커트 5점 척도 점수는 기사의 미묘한 뉘앙스를 합리적으로 반영해야 합니다.

---
*이하 내용은 React/FastAPI 기반의 "뉴스 요약·감성분석 웹/앱 v2.1" 기준으로 재정의된 성공 기준입니다.*

## 3.3. 프로젝트 성공 기준 (v2.1)

1.  **성능 (응답 시간)**:
    -   **측정 항목**: 사용자가 프론트엔드에서 "분석 실행" 버튼을 클릭한 후, 최종 분석 결과(요약, 감성)가 화면에 표시되기까지의 총 소요 시간.
    -   **성공 기준**: 단일 URL 분석 요청에 대해 **15초 이내**에 결과를 표시하는 것을 목표로 합니다.
    -   **비고**: 이 시간은 외부 요인(LLM API 응답 속도, 대상 웹사이트 응답 속도)에 크게 의존하지만, 백엔드 내부 처리 시간을 최소화하여 사용자 경험을 개선합니다.

2.  **사용자 경험 (UX)**:
    -   **측정 항목**: 사용자가 기능을 사용하는 데 있어 직관성과 편의성.
    -   **성공 기준**:
        -   사용자가 별도의 가이드 없이도 URL 및 API 키 입력, 분석 실행, 결과 확인 등 주요 흐름을 막힘없이 진행할 수 있어야 합니다.
        -   분석 대기 중에는 로딩 상태(스피너 등)가 명확히 표시되어야 합니다.
        -   API 키 오류, 잘못된 URL 등 예상 가능한 오류 발생 시, 원인을 파악할 수 있는 명확한 메시지가 사용자에게 제공되어야 합니다.

3.  **안정성 및 에러 처리**:
    -   **측정 항목**: 주요 기능(본문 추출, 요약, 감성 분석) 실패율 및 대체 흐름.
    -   **성공 기준**:
        -   백엔드에서 특정 서비스(예: 요약)가 실패하더라도, API 요청 자체가 5xx 에러로 중단되는 대신, 해당 필드를 "요약 실패"와 같은 메시지로 채운 200 OK 응답을 반환해야 합니다.
        -   프론트엔드는 이렇게 부분적으로 실패한 결과를 정상적으로 화면에 표시하여, 사용자가 어떤 부분에서 문제가 발생했는지 인지할 수 있도록 해야 합니다.
        
4.  **확장성**:
    -   **측정 항목**: 새로운 기능(예: 키워드 검색) 추가 시 필요한 코드 변경의 범위.
    -   **성공 기준**: 프론트엔드와 백엔드의 역할이 명확히 분리되어 있어, 백엔드 API 규격만 맞추면 프론트엔드 코드의 수정 없이도 새로운 LLM 공급자를 추가할 수 있는 등, 기능 확장이 용이해야 합니다.

5.  **키워드 검색 기능**:
    -   **측정 항목**: 키워드 기반 뉴스 검색 기능의 정확성 및 사용성.
    -   **성공 기준**:
        -   사용자가 입력한 키워드에 대해 NewsAPI를 통해 관련성 높은 뉴스 기사를 정확하게 검색하여 표시해야 합니다.
        -   검색 결과 목록에서 뉴스 기사를 선택하면 해당 URL이 자동으로 분석 필드에 채워져야 합니다.

6.  **API 키 처리 유연성**:
    -   **측정 항목**: 백엔드 환경 변수 및 UI를 통한 API 키 입력의 정상 작동 여부.
    -   **성공 기준**:
        -   백엔드에 API 키 환경 변수(`NEWS_API_KEY`, `LLM_API_KEY` 등)가 설정되어 있으면 UI 입력보다 우선하여 해당 키를 사용해야 합니다.
        -   환경 변수가 설정되지 않은 경우, UI를 통해 입력된 키가 정상적으로 백엔드로 전달되어 분석 및 검색 작업에 사용되어야 합니다.
